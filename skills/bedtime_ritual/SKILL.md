# 星光时间 · Bedtime Rose, Thorn, Bud

## 概述

每天睡前，星仔作为"主持人"引导家长和孩子各自分享当天的经历和感受。分享结束后，星仔将双方的故事融合，生成一张温馨的插画，作为次日家长日记的头图。

**核心价值：**
- 对孩子：培养情绪觉察、表达能力、安全感
- 对家长：感受到真实的亲子连接，获得可保存的"家庭记忆"
- 对产品：每日生成独一无二的插画 → 高频打开日记 → 用户粘性

---

## 前置条件

| 条件 | 来源 |
|------|------|
| 孩子在线（有对话 session） | 系统检测 |

| 当天尚未执行过该 Skill | 状态存储 |
| 孩子画像已建立（年龄、兴趣） | Onboarding 数据 |

## 执行流程

### Phase 0: 邀请

**目标：** 通知家长 + 唤起孩子对活动的期待

```
→ 推送家长端：
  "今晚的星光时间要开始啦 🌟 
   准备好和糯糯一起分享今天的故事吗？
   [开始] [今天跳过]"

→ 如果家长点击 [开始]：
  星仔对孩子说：
  "糯糯！星光时间到啦～✨ 
   今天我们和妈妈一起来玩一个好玩的游戏——
   分享今天的玫瑰🌹、刺🌵和小芽🌱！"

→ 如果家长点击 [今天跳过]：
  记录跳过，Skill 不执行
  星仔对孩子正常聊天或进入睡前故事

→ 如果家长不在（异地模式 / 超时未响应）：
  星仔只引导孩子分享，结束后整理发给家长
  家长可以稍后补充自己的分享
```

**工具调用：** `push_notifier.send(parent_id, invitation_card)`

---

### Phase 1: 孩子的🌹 Rose

**目标：** 引导孩子分享今天最开心/最棒的事

```
星仔："糯糯，你今天的玫瑰是什么呀？
      就是今天最让你开心的一件事！🌹"

等待孩子回答...

星仔（回应）："哇，[复述孩子说的]！听起来真的好棒！"
```

**引导策略：**
- 如果孩子说"不知道" → "那今天有没有什么好玩的事？比如在幼儿园？"
- 如果孩子说得很短 → "然后呢？可以跟我多说一点吗？"
- 如果孩子跑题 → 不打断，让孩子自由表达，记录关键词

**数据记录：** `child_rose: { raw_text, keywords, emotion }`

---

### Phase 2: 孩子的🌵 Thorn（可选）

**目标：** 给孩子一个安全的空间说出不开心的事

```
星仔："那今天有没有不太开心的事呀？就像小刺🌵一样扎扎的？
      没有的话也没关系哦～"

→ 如果孩子说了 → 共情："那确实有点不舒服呢。不过你告诉我了，
                       我们一起把它放进担心盒子里好不好？"

→ 如果孩子说没有 → "太好了！今天一天都很顺利呀！"
```

**重要规则：**
- ❌ 不追问细节、不教育、不说"下次你应该..."
- ✅ 只做倾听和共情
- ❌ 孩子说到严重问题（被欺负等）不在此环节处理，记录后推送给家长

**数据记录：** `child_thorn: { raw_text, emotion, severity }`

---

### Phase 3: 孩子的🌱 Bud

**目标：** 建立对明天的期待感

```
星仔："最后一个！你的小芽🌱是什么？
      就是你明天最期待的事！"

→ 孩子回答...

星仔："好棒！明天一定会很精彩的！"
```

**数据记录：** `child_bud: { raw_text }`

---

### Phase 4: 家长的🌹 Rose

**目标：** 家长自己分享，让孩子感受到"爸妈也是人"

```
星仔："轮到妈妈了！妈妈，你今天的玫瑰🌹是什么呀？"

等待家长回答...

星仔（对孩子说）："妈妈今天也过了好棒的一天呢！"
```

**设计意图：**
- 研究表明家长先分享自己的事，会让孩子觉得"我们是平等的"
- 但放在孩子之后是因为：孩子注意力有限，先让孩子说完

**数据记录：** `parent_rose: { raw_text }`

---

### Phase 5: 家长的"我注意到..."

**目标：** 引导家长给出具体的、基于观察的正向反馈

```
星仔："妈妈，今天有没有注意到糯糯做了什么让你开心的事呀？
      跟她说说吧～"

→ 家长说...

星仔（对孩子）："你听到了吗？妈妈一直都注意到你呢 ☺️"
```

**为什么这一步很重要：**
- "我注意到你今天主动收拾了玩具" 比 "你真棒" 有效 10 倍
- 帮助家长学会"观察式表达"——很多家长不知道怎么夸孩子

**数据记录：** `parent_observation: { raw_text }`

---

### Phase 6: 收尾仪式

```
星仔："今天的星光时间好温暖呀～
      糯糯今天 [一句话总结 Rose]，
      妈妈今天也 [一句话总结]，
      你们俩都好棒！
      
      来，让我把今天的故事画下来...✨"

（生成图片中...播放轻柔音乐/等待动画）

星仔："画好啦！你们看——
      [展示生成的图片]
      这是你们今天的星光画✨
      
      妈妈来说晚安吧～"

家长："晚安宝贝，妈妈爱你"

星仔："晚安糯糯，明天见！🌙"
```

---

### Phase 7: 后台处理

**在对话结束后异步执行：**

```
1. 整合数据 → 生成图片 Prompt
   输入：child_rose + parent_rose + child_bud + 孩子兴趣 + 时间/季节
   
2. 调用图片生成
   工具：image_generator.generate(prompt, style="watercolor_children_book")
   
3. 生成日记条目
   标题：自动生成（如"糯糯和妈妈的星光故事 · 4月3日"）
   头图：生成的插画
   正文：Rose/Thorn/Bud 的结构化摘要 + 家长的观察
   
4. 推送给家长
   工具：push_notifier.send(parent_id, journal_card)
   "今天的星光画已经画好啦 🎨
    [查看日记]  [保存图片]  [分享]"
   
5. 更新记忆
   工具：memory_store.save({
     type: "bedtime_ritual",
     date: today,
     child_rose, child_thorn, child_bud,
     parent_rose, parent_observation,
     image_url
   })
```

## 图片生成策略

### Prompt 组装逻辑

```
基础风格：
  "A warm watercolor illustration in children's book style, 
   soft pastel colors, gentle lighting, dreamy atmosphere"

主题元素（从分享内容提取）：
  child_rose: "去公园滑滑梯" → "a child sliding down a colorful slide in a sunny park"
  parent_rose: "做了好吃的蛋糕" → "a beautiful cake with candles"
  child_bud: "明天想去看鱼" → "small goldfish swimming around"

融合 Prompt：
  "A warm watercolor illustration in children's book style.
   A happy child sliding down a rainbow slide in a sunny park, 
   with a beautiful cake on a picnic blanket nearby,
   and small goldfish swimming in a pond in the background.
   Soft pastel colors, gentle lighting, dreamy atmosphere.
   No text. 16:9 ratio."
```

### 风格一致性

| 参数 | 固定值 | 原因 |
|------|--------|------|
| 风格 | 水彩儿童绘本风 | 温暖、不写实、有艺术感 |
| 色调 | 暖色柔和 | 与梨谱设计系统一致 |
| 比例 | 16:9 | 适合做日记头图 |
| 画面 | 没有文字 | 干净，家长可以叠加 |
| 人物 | 不画真实人脸 | 避免恐怖谷效应，用可爱简笔代替 |

### 特殊情况

| 场景 | 处理 |
|------|------|
| 孩子说了 Thorn | 图片**不**画 Thorn 内容，只画 Rose 和 Bud |
| 内容太抽象 | 用孩子画像中的兴趣元素补充（恐龙、太空...） |
| 敏感内容 | 过滤后用通用温馨场景兜底 |

## 成功判定

| 指标 | 判定标准 |
|------|---------|
| ✅ 完成 | 孩子至少说了 Rose；图片成功生成 |
| 🟡 部分完成 | 孩子说了但图片生成失败 → 用默认温馨图 |
| ⏭️ 跳过 | 家长选择跳过 / 孩子明确不想玩 |
| ❌ 失败 | 技术故障 → 记录错误，明天重试 |

---

## 失败处理

| 失败场景 | 处理 |
|---------|------|
| 家长不在场 | 切换异地模式：只引导孩子，结果发家长，家长稍后补充 |
| 孩子不想玩 | "没关系！那我们讲个睡前故事好不好？" → 不强制 |
| 孩子中途跑掉 | 保存已有内容，用已有内容生成图片 |
| 图片生成失败 | 用预制的温馨默认图 + "今天的画明天补给你哦～" |
| 家长超时未回复邀请 | 10分钟后自动开始孩子独立模式 |

## 变体和进化

### 按年龄调整

| 年龄 | 调整 |
|------|------|
| 2-3岁 | 只问 Rose（"今天好玩吗？"），不问 Thorn/Bud |
| 4-5岁 | 完整 Rose/Thorn/Bud，用简单比喻辅助 |
| 6-7岁 | 可以加 Gratitude 环节（"今天想感谢谁？"）|

### 每周特别版

| 周期 | 变化 | 目的 |
|------|------|------|
| 每周五 | "这周最大的玫瑰🌹是什么？" | 培养回顾习惯 |
| 每月底 | 生成月度拼图（30张日图合成） | 增强收藏价值 |
| 节日 | 特别主题（中秋→月亮主题画） | 仪式感 |

### 跟成长目标联动

```
如果当前有 Goal "认识情绪" 正在进行：
  → Thorn 环节可以多引导一步：
    "这个扎扎的感觉是什么颜色的？"
    （自然地融入情绪认知目标，而不是专门"上课"）
```

## 产品闭环

```
睡前对话（输入）
    ↓
生成插画（产出 1 → 即时展示，增强仪式感）
    ↓
次日日记头图（产出 2 → 家长打开日记的理由）
    ↓
日记内含 Rose/Thorn/Bud 摘要（产出 3 → 家长了解孩子）
    ↓
家长保存/分享（产出 4 → 社交传播 / 留住记忆）
    ↓
长期：月度/年度回顾画册（产出 5 → 付费点？）
```
